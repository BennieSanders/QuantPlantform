# Quant Platform 结项演示手册

## 1. 演示前检查

在项目根目录运行：

```bash
./scripts/restart_demo.sh
```

该命令会自动停止旧服务、重新启动后端 `8000` 和前端 `5173`，并执行一次自检。浏览器访问 `http://127.0.0.1:5173`。

如果希望使用前台方式保持终端运行，也可以执行：

```bash
./scripts/demo.sh
```

另开一个终端可再次执行：

```bash
./scripts/check_demo.sh
```

只有全部显示 `[OK]` 后再开始演示。默认端口不是 `5174`；如需使用 `5174`，请运行 `FRONTEND_PORT=5174 ./scripts/restart_demo.sh`。

建议提前完成一次注册、登录，并保留至少一条成功回测和一条 Gemini 分析记录。

## 2. 演示顺序（8-10 分钟）

1. **登录与系统定位（1 分钟）**：展示独立注册/登录页，说明登录后数据按用户隔离。
2. **实时行情观察（2 分钟）**：选择 BTCUSDT 和 1m，展示当日实时 K 线、成交量、MA7/MA25 和数据健康状态。
3. **策略回测（3 分钟）**：设置时间段、周期、初始资金和策略参数，提交异步任务，展示收益曲线、交易标记和风险指标。
4. **智能分析（2 分钟）**：对刚完成的回测使用 Google Gemini，展示 provider、评分、优势、风险、执行计划和建议参数。
5. **工程化总结（1-2 分钟）**：说明前后端分离、数据库迁移、异步任务、自动化测试和本地回退机制。

## 3. 演示话术重点

- 分析模块使用回测结构化指标作为上下文，输出可保存、可追溯、可切换 provider 的建议。
- 实时行情同时完成拉取、持久化、质量检查和图表观察。
- 回测与分析形成闭环：行情数据 -> 策略回测 -> 风险指标 -> 分析建议 -> 参数迭代。
- 本地引擎用于网络或额度异常时的可复现兜底。

## 4. 故障预案

### Gemini 无法连接

- 切换到“本地引擎”。
- 打开历史分析，展示此前保存的 Gemini 结果。
- provider 和回退原因会明确展示，便于区分外部模型和本地引擎结果。

### Binance 无法连接

- 不再点击同步，直接展示数据库中已保存的近期 K 线。
- 回测可使用数据库数据；覆盖不足时会回退到版本库中的 CSV 样例。

### 新回测耗时或失败

- 打开回测历史中的成功记录。
- 在任务中心展示任务状态、失败原因和重试能力。

### 端口被占用

```bash
./scripts/restart_demo.sh
```

### 页面缓存旧版本

- 浏览器强制刷新。
- 确认地址是 `http://127.0.0.1:5173`。

## 5. 演示结束

在运行 `demo.sh` 的终端按 `Ctrl+C`。如果使用后台方式启动，则运行 `./scripts/stop_demo.sh`。
